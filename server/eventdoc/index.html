Emit:

Events Documentation
Events to listen:
notifcation event
This event ("notifcation") has to be listened on the user's end to get notifications ('Meeting Updates').
Output Data
{
notificationId: String,
notificationType: String,
userId: String,
meeting: Object
}
reminder event
This event ("reminder") has to be listened on the user's end to get the reminder of an approaching meeting plan.
Output Data
{
meetingId: String,
userId: String,
purpose: String,
date: {
day: Number,
month: Number,
year: Number
},
startTime: {
hour: Number,
minute: Number
},
endTime: {
hour: Number,
minute: Number
},
place: String,
createdAt: Date,
updatedAt: Date,
createdBy: {
adminId: String,
adminName: String
},
updatedBy: {
adminId: String,
adminName: String
}
}
logout event
This event ("logout") has to be listened on the user's end to know that some user has been logged out.
Output Data
{
userId: String
}
Events to emit:
set-user event
This event ("set-user") has to be emitted by the user's end to let the server set the userId so that private events can be emitted for the specific user.
Data To Be Emitted
{
userId: String
}
snooze event
This event ("snooze") has to be emitted by the user's end to let the server send another reminder after 5 seconds.
Data To Be Emitted
{
meetingId: String,
userId: String,
purpose: String,
date: {
day: Number,
month: Number,
year: Number
},
startTime: {
hour: Number,
minute: Number
},
endTime: {
hour: Number,
minute: Number
},
place: String,
createdAt: Date,
updatedAt: Date,
createdBy: {
adminId: String,
adminName: String
},
updatedBy: {
adminId: String,
adminName: String
}
}
